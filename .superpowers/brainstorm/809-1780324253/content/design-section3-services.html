設計第三節：Service 層

TPS.Nexus.Kanban.Services 專案內容

服務分組總覽

Services/
Map/　　　← 地圖匯入與解析
DataSource/　← 資料來源適配器
Layout/　　← 版本草稿/發布
Equipment/　← 設備與看板管理
Alarm/　　← 警報偵測與推送
Icon/　　　← 圖示上傳管理
Hubs/　　　← SignalR Hub

地圖匯入（Map/）

MapImportService : IMapImportService
依 MapFormatType 路由至對應 Parser
ImageMapHandler　← PNG / JPG
儲存原始圖檔，產生縮圖
SvgMapParser　　← SVG
解析 SVG 結構，萃取可互動圖層
DxfMapParser　　← DXF / CAD
使用 netDxf 函式庫轉換為 SVG
JsonXmlCoordParser　← JSON / XML 座標定義
解析設備位置座標，建立 FactoryMap 物件

資料來源適配器（DataSource/）

DataSourceService : IDataSourceService
依 DataSourceType 路由，統一回傳 DataResult
SqlDataAdapter　← 使用 IDbConnectionFactory
執行 SQL 查詢，支援參數化查詢防 Injection
CsvDataAdapter
使用 CsvHelper 解析，支援自訂分隔符
JsonDataAdapter
System.Text.Json 解析，支援 JSONPath 取值
XmlDataAdapter
System.Xml 解析，支援 XPath 取值

佈局版本管理（Layout/）

LayoutService : ILayoutService
SaveDraftAsync　→ 儲存草稿（Status = Draft）
PublishAsync　　→ 草稿升版（Status = Published），舊版本 → Archived
GetVersionHistoryAsync → 取得所有版本清單
RollbackAsync　→ 將指定版本重新 Published

警報偵測與推送（Alarm/ + Hubs/）

AlarmService : IAlarmService
EvaluateAsync　→ 比對 AlarmRule 條件，觸發時呼叫 Hub
GetActiveAlarmsAsync → 取得目前未解除警報清單
KanbanAlarmHub : Hub ← SignalR
SendAlarmAsync(equipmentId, alarmLevel, message)
客戶端監聽 "ReceiveAlarm" 事件即時更新畫面

模組註冊（KanbanModuleRegistrar）

KanbanModuleRegistrar : IModuleRegistrar
Register(IServiceCollection services)
services.AddScoped<IMapImportService, MapImportService>()
services.AddScoped<IDataSourceService, DataSourceService>()
services.AddScoped<ILayoutService, LayoutService>()
services.AddScoped<IAlarmService, AlarmService>()
services.AddScoped<IIconUploadService, IconUploadService>()
services.AddSignalR() → KanbanAlarmHub
