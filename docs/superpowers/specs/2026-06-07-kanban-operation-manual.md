## 1. 系統概述

TPS.Nexus 工廠看板系統讓操作員與管理者能在廠區樓層圖上即時掌握設備運行狀態。

**主要功能**：
- 廠區平面圖上顯示各設備即時狀態
- 多地圖自動輪播（巡視模式）
- 點擊設備查看詳細參數、工單與警報紀錄
- 管理者可拖曳設備位置、發布佈局版本
- 異常狀態即時警報推送（SignalR）

**系統路由**：

| 路由 | 說明 |
|---|---|
| `/kanban/1`、`/kanban/2`… | 各廠區看板地圖 |
| `/kanban/settings` | 設定管理（需 KANBAN_SETTINGS 權限） |

## 3. 看板地圖 — 基本操作

### 3.1 切換地圖

**方式一：工具列下拉選單**
1. 點擊工具列左側的地圖名稱下拉
2. 從清單中選擇目標廠區
3. 頁面自動切換，URL 更新為 `/kanban/{新地圖 ID}`

**方式二：頂部「🗺 看板地圖」連結**
- 點擊後跳回預設地圖（上次使用的地圖）
- 系統透過 localStorage 記憶上次選取的地圖

> **提示**：系統會記住您上次查看的地圖。下次進入系統時自動跳轉至該地圖。

### 3.2 設備狀態燈號

地圖上每個設備 Widget 左下角有狀態燈：

| 燈號 | 顏色 | 意義 |
|---|---|---|
| ● | 綠色（發光） | 設備運行中 |
| ● | 橘色（發光） | 待機 |
| ● | 灰色 | 停機 |
| ● | 紅色（閃爍） | 警報中 |

警報狀態下，Widget 整體背景轉為深紅色，並在右上角顯示 **!** 紅色角標。

## 4. 地圖輪播

輪播功能讓多廠區地圖自動循環顯示，適用於大螢幕巡視場景。

### 4.1 輪播說明

當系統設定多張地圖均啟用輪播（`CarouselEnabled = true`），地圖會依序自動切換。

**倒數徽章**（右下角）：
```
┌─────────────────────────┐
│  ⏱ 8 秒後切換   ⏸ 暫停 │
└─────────────────────────┘
```

| 顯示 | 意義 |
|---|---|
| ⏱ N 秒後切換 | 輪播進行中，N 秒後跳到下一張地圖 |
| ⏸ 輪播已暫停 + ▶ 繼續 | 輪播已暫停 |

### 4.2 暫停 / 繼續輪播

**手動暫停**：點擊徽章右側「**⏸ 暫停**」按鈕

**繼續輪播**：點擊「**▶ 繼續**」按鈕，從剩餘秒數繼續倒數

**自動暫停的場景**：

| 場景 | 暫停原因 |
|---|---|
| 開啟版本面板（📋 版本） | 避免版本操作中途切換地圖 |
| 進入編輯模式 | 編輯時停止輪播 |

**自動恢復的場景**：

| 場景 | 恢復說明 |
|---|---|
| 關閉版本面板 | 從暫停時的剩餘秒數繼續 |
| 完成編輯（發布或按「✅ 完成編輯」） | 重新啟動輪播 |
| 地圖切換（輪播觸發的導覽） | 自動以該地圖的設定秒數重啟 |

> **注意**：切換至新地圖時，暫停狀態會自動重置為「未暫停」。

## 5. 全螢幕模式

### 5.1 進入全螢幕

點擊工具列右側「**⛶**」按鈕，地圖區域擴展至全螢幕。

### 5.2 全螢幕下的功能

全螢幕模式下，所有功能正常運作：
- ✅ 輪播倒數徽章（右下角可見）
- ✅ 暫停/繼續輪播按鈕
- ✅ 設備 Widget 點擊
- ✅ 警報角標

### 5.3 離開全螢幕

- 按鍵盤 **Esc**，或
- 再次點擊「**⛶**」按鈕

## 7. 編輯模式

> **所需權限**：KANBAN_EDIT（編輯）、KANBAN_PUBLISH（發布）

編輯模式讓管理者調整設備在地圖上的位置，並管理佈局版本。

### 7.1 進入編輯模式

1. 點擊工具列「**✏️ 編輯**」按鈕
2. 若存在**未發布草稿**，系統會詢問：

```
┌──────────────────────────────────────────────────┐
│  找到先前儲存的草稿（v3），是否繼續上次的編輯工作？  │
│                                                  │
│  [ 繼續編輯 ]          [ 重新開始 ]               │
└──────────────────────────────────────────────────┘
```

| 選項 | 行為 |
|---|---|
| **繼續編輯** | 載入草稿 v3 的佈局繼續修改 |
| **重新開始** | 以目前發布版本為基礎建立全新草稿 |
| 關閉對話框（X） | 取消，不進入編輯模式 |

3. 進入後：
   - 輪播**自動暫停停止**（不顯示徽章）
   - 設備 Widget 邊框變為**虛線**，可拖曳

### 7.2 拖曳設備

1. 在編輯模式中，直接拖曳設備 Widget 至新位置
2. 放開後，位置**即時儲存**至草稿

> **提示**：系統自動儲存每次拖曳的位置，不需要手動儲存每一步。

### 7.3 儲存草稿

點擊工具列「**💾 存草稿**」：
- 將目前佈局儲存為草稿版本
- 草稿不影響其他使用者看到的正式版本
- 可多次儲存草稿（版本號遞增）

### 7.4 發布

點擊工具列「**🚀 發布**」：
- 將目前草稿設為**正式發布版本**
- 舊的正式版本自動封存（Archived）
- 所有使用者立即看到新佈局
- 離開編輯模式，輪播自動重啟

### 7.5 離開編輯模式（不發布）

點擊「**✅ 完成編輯**」：
- 退出編輯模式
- **草稿保留**，下次進入編輯模式時可選擇繼續
- 輪播自動重啟

## 8. 版本管理

### 8.1 開啟版本面板

點擊工具列「**📋 版本**」：
- 面板從右側**疊加**於地圖上（地圖不縮小）
- 輪播自動暫停
- 顯示目前地圖的所有佈局版本

### 8.2 版本面板說明

```
┌────────────────────────────────┐
│  📋 版本紀錄                   │
│ ────────────────────────────── │
│  v4  [已發布]  06/05 14:23     │
│      admin                     │
│                                │
│  v3  [封存]    06/04 09:15     │
│      admin             [回溯]  │
│                                │
│  v2  [封存]    06/03 11:42     │
│      operator          [回溯]  │
│                                │
│  v1  [草稿]    06/07 10:00     │
│      admin                     │
└────────────────────────────────┘
```

| 狀態標籤 | 顏色 | 說明 |
|---|---|---|
| **已發布** | 綠色 | 目前正式版本（同時只有一個） |
| **封存** | 灰色 | 歷史版本，可回溯 |
| **草稿** | 藍色 | 未發布的進行中版本 |

### 8.3 回溯版本

1. 在版本面板找到目標封存版本
2. 點擊該版本右側「**回溯**」按鈕
3. 系統將該版本設為新的「已發布」版本
4. 原「已發布」版本自動封存

> **注意**：回溯後，所有使用者立即看到回溯後的佈局。此操作無法直接撤銷，但可再次回溯至任意封存版本。

### 8.4 關閉版本面板

再次點擊工具列「**📋 版本**」，輪播自動恢復。

## 9. 即時警報

### 9.1 警報通知

警報條件觸發時（由後台評估），畫面**右下角自動彈出通知**：

```
┌──────────────────────────────────────────┐
│  ⚠ 警報 [Critical]                       │
│  CNC-01: temp > 80 (actual: 85.3)         │
└──────────────────────────────────────────┘
```

通知持續 5 秒後自動消失。

| 警報等級 | 通知顏色 | Widget 狀態 |
|---|---|---|
| **Critical**（緊急） | 紅色 | 背景深紅 + 角標 + 紅燈閃爍 |
| **Warning**（警告） | 橘色 | 背景深橘 + 角標 + 紅燈閃爍 |
| **Info**（資訊） | 藍色 | 角標 |

### 9.2 多人共享

系統使用 **SignalR** 即時推送，所有已連線的使用者**同步收到**相同警報通知，不需要重新整理頁面。

## 10. 設定管理

> **所需權限**：KANBAN_SETTINGS

前往路由 `/kanban/settings` 或點擊頂部導覽列「**⚙ 設定管理**」。

設定管理包含 4 個頁籤：

---

### 10.1 設備管理

管理廠區設備的基本資料。

#### 新增設備

1. 點擊「**+ 新增設備**」
2. 填寫以下欄位：

| 欄位 | 說明 | 必填 |
|---|---|---|
| **名稱** | 設備顯示名稱（如「CNC-01」） | ✅ |
| **標籤** | 設備代號或識別碼 | — |
| **說明** | 設備描述 | — |
| **圖示類型** | CssClass（Font Awesome）或 CustomImage | ✅ |
| **圖示值** | CSS class 名稱（如 `fa-solid fa-gear`）或圖片路徑 | — |

3. 點擊「**儲存**」

#### 編輯設備

1. 在設備列表中點擊「**編輯**」
2. 修改欄位後點擊「**儲存**」

#### 刪除設備

點擊「**刪除**」，確認後設備從系統移除。

> **注意**：刪除設備前，請確認該設備已從所有地圖佈局中移除，否則可能造成顯示異常。

---

### 10.2 資料來源

設定設備參數的資料來源，支援 SQL、CSV、JSON、XML 四種類型。

#### 新增資料來源

1. 點擊「**+ 新增資料來源**」
2. 填寫以下欄位：

| 欄位 | 說明 |
|---|---|
| **名稱** | 資料來源識別名稱 |
| **類型** | SQL / CSV / JSON / XML |
| **查詢/路徑** | SQL 查詢語句，或 CSV/JSON/XML 檔案路徑 |
| **連線字串** | 僅 SQL 類型需要填寫 |
| **參數** | JSON 格式的查詢參數，例如 `{"machineId": 1}` |

#### SQL 資料來源範例

```
類型: SQL
查詢: SELECT speed, temp, yield FROM machine_data WHERE id = @machineId
參數: {"machineId": 1}
```

> **安全提示**：SQL 查詢強制使用參數化查詢，防止 SQL Injection。請勿在查詢語句中直接拼入變數。

#### CSV 資料來源範例

```
類型: CSV
路徑: /data/machine_daily.csv
```
CSV 第一列為欄位名稱，讀取第一筆資料列作為最新數值。

---

### 10.3 圖示上傳

上傳自訂設備圖示，供設備管理中「**CustomImage**」類型使用。

**支援格式**：PNG、JPG、JPEG、SVG  
**檔案大小上限**：5 MB

1. 點擊「選擇檔案」
2. 選取圖示檔案
3. 點擊「**上傳**」
4. 系統回傳圖片路徑（如 `/module-assets/TPS.Nexus.Kanban/images/equipment-icons/xxxx.png`）
5. 將此路徑填入設備管理的「圖示值」欄位

---

### 10.4 地圖管理

管理廠區底圖，支援匯入、查看和刪除。

#### 匯入新地圖

1. 選擇地圖格式：

| 格式 | 說明 |
|---|---|
| **PNG 圖片** | 廠區平面圖截圖 |
| **JPG 圖片** | 同上 |
| **SVG 向量圖** | 可縮放向量圖，線條清晰 |
| **DXF 工程圖** | AutoCAD 格式，自動轉換為 SVG |
| **JSON 座標定義** | 純數據定義設備位置，無底圖 |
| **XML 座標定義** | 同上，XML 格式 |

2. 點擊「**選擇檔案**」並選取地圖檔案
3. 點擊「**上傳並匯入**」
4. 匯入成功後，地圖出現在「已匯入地圖」列表

#### 已匯入地圖列表

列表顯示以下欄位：

| 欄位 | 說明 |
|---|---|
| **名稱** | 地圖名稱（取自上傳的檔案名） |
| **格式** | PNG / JPG / SVG / DXF / JsonCoord / XmlCoord |
| **建立時間** | 匯入日期 |
| **版本** | 管理版本標示（如 v1、v2） |

#### 刪除地圖

點擊地圖列表右側「**刪除**」按鈕移除地圖。

> **注意**：刪除地圖後，該地圖的所有佈局版本仍保留在資料庫，但地圖頁面無法存取。

## 11. 權限說明

系統使用功能權限碼控制各項操作的可見性與可用性：

| 權限碼 | 可執行操作 | 適用角色 |
|---|---|---|
| **KANBAN_VIEW** | 查看看板地圖、設備狀態、版本面板（唯讀） | 所有使用者 |
| **KANBAN_EDIT** | 進入編輯模式、拖曳設備、存草稿 | 管理者 |
| **KANBAN_PUBLISH** | 發布草稿、回溯版本 | 資深管理者 |
| **KANBAN_SETTINGS** | 設備管理、資料來源、圖示上傳、地圖管理 | 系統管理員 |

**無權限時的顯示**：
- 無 VIEW 權限：顯示「無查看權限」訊息，無法看到地圖
- 無 EDIT 權限：工具列不顯示「編輯」按鈕
- 無 PUBLISH 權限：工具列不顯示「發布」按鈕，版本面板不顯示「回溯」按鈕
- 無 SETTINGS 權限：導覽「設定管理」顯示「無設定權限」訊息

## 12. 常見問題

### Q1：切換地圖後，看板地圖按鈕變成藍色？

**正常行為**：「🗺 看板地圖」連結在任何 `/kanban/{n}` 頁面下均保持**白色**（active 狀態）。若顯示藍色，請重新整理頁面，或確認系統版本是否為 v1.3 以上。

---

### Q2：全螢幕模式下，輪播倒數計時消失了？

**舊版本問題**，v1.3 已修正。倒數徽章現在位於地圖容器內部，全螢幕模式下正常顯示。若仍遇到此問題，請清除瀏覽器快取並重新整理。

---

### Q3：版本面板開啟後輪播停了，關掉後沒有繼續？

**操作步驟**：
1. 開啟版本面板 → 輪播暫停（正常行為）
2. 再次點擊「📋 版本」關閉面板 → 輪播自動從暫停前的秒數繼續

若關閉後未恢復，請確認您沒有同時進入了編輯模式（編輯模式下輪播停止，不顯示徽章）。

---

### Q4：我拖曳了設備但忘了發布，其他人看到的是舊版嗎？

**是的**。草稿不會影響正式版本。只有點擊「**🚀 發布**」後，所有使用者才會看到新佈局。未發布的草稿僅對您可見（在編輯模式中）。

---

### Q5：我誤發布了錯誤的佈局，如何還原？

使用**版本回溯**功能：
1. 點擊「📋 版本」開啟版本面板
2. 找到正確的封存版本
3. 點擊「**回溯**」

系統會立即將該版本設為新的正式版本，所有使用者即時看到還原後的佈局。

---

### Q6：警報通知出現後就消失了，在哪裡查看歷史警報？

點擊任意設備 Widget，在詳情抽屜中選擇「**警報歷史**」頁籤，可查看該設備的所有歷史警報紀錄。

---

### Q7：地圖下拉選單的地圖名稱顯示不正確？

地圖名稱取自匯入時的檔案名稱。如需修改，請前往「**設定管理 → 地圖管理**」，刪除後重新以正確命名的檔案匯入。

---

### Q8：系統連線中斷後，警報還會推送嗎？

SignalR 連線中斷時，警報推送會暫停。系統在頁面初始化時建立連線，若連線失敗會顯示黃色警告通知「**警報連線失敗**」。此時請重新整理頁面以重建連線。
